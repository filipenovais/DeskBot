DeskBot Setup

Get your voice-activated AI assistant running in minutes

0
Prerequisites

Make sure you have the following installed:

Git

For cloning the repository and running bash scripts on Windows.

Download Git

Docker Desktop

Required for running the database service container.

Download Docker Desktop

Conda (Miniconda or Anaconda)

Python environment manager for dependencies.

Download Miniconda (recommended) or Download Anaconda

Groq API Key Optional

Get free API access for LLM, STT (Whisper), and TTS services.

Get Groq API Key (Free)

Alternatively, run fully local with Ollama (no API key needed).

1
Run the Install Script

Open your terminal and navigate to the project folder:

Windows (Git Bash) or Linux/macOS (Terminal)
# Make the script executable chmod +x install.sh # Run the installer ./install.sh
Want to run fully local? Add the --ollama flag to also start the Ollama service:
./install.sh --ollama

The script will:

  • Check for required dependencies
  • Start the database service (Docker)
  • Optionally start Ollama service
  • Create the deskbot_env conda environment
2
Start DeskBot

Activate the environment and run:

conda activate deskbot_env python run.py

DeskBot will appear in your system tray. The conversations window opens automatically on startup.

3
Configure Services

Right-click the tray icon and select Settings to configure:

  • LLM Provider - Choose Groq, OpenAI, Anthropic, or Ollama
  • API Base URL - Set the endpoint for your provider
  • API Key - Enter your API key (not needed for Ollama)
  • STT/TTS - Configure speech services
  • Hotkeys - Customize keyboard shortcuts
  • Microphone - Select your input device
Service Status: Check the colored dots in the conversations window:

Service connected Service offline
4
Start Using

Default hotkeys:

  • Shift+F12 - Hold to talk (push-to-talk)
  • Shift+F2 - Create & open new conversation
  • Shift+F3 - Open current conversation
  • Shift+F11 - Create new conversation
  • Shift+F1 - Open conversations window

All conversations are stored locally in the Docker database container.
